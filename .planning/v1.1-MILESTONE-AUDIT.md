## Gap Details

### PARTIAL: EVTS-01, EVTS-02, EVTS-03 — Event Lifecycle Not Integrated

The `Event` class in `backend/schemas/events.py` implements a full lifecycle state machine (created → active → spreading → expired), propagation tracking (`heard_by`), and tick-based expiry (`is_expired()`). All 17 unit tests pass.

**However:** No runtime code uses `Event` objects. `engine.inject_event()` writes events directly to ChromaDB via `add_memory()`. The Event lifecycle is a schema artifact — it exists and is tested but is never exercised in the live simulation.

**Impact:** Low. Events still work correctly for users (injected events reach agents, agents react). The lifecycle tracking is a design-forward feature that was built but not wired in.

### PARTIAL: ARCH-02 — Engine Bypasses 2 of 3 Agent Wrappers

The `Agent` class has `perceive()`, `decide()`, `converse()`, and `reflect()` wrapper methods. The engine routes conversations through `agent.converse()` but calls `perceive()` and `decide_action()` directly from imported modules.

**Impact:** Low. Functionally identical — the wrapper methods are thin delegates. The OOP encapsulation goal is partially realized.

### PARTIAL: VIS-02 — Activity Text Removed

Phase 8 Plan 01 added 20px name labels with pill backgrounds and 16px activity text. Commit `686ddf7` (fix(09)) subsequently removed the activity text and pill backgrounds, simplifying agent sprites to just name labels at 20px.

**Impact:** Medium. Agent names are readable (requirement partially met). But activity text is no longer visible on the map — users must click an agent to see what they're doing. The name label uses dark navy fill which may be hard to read on dark sector backgrounds.
