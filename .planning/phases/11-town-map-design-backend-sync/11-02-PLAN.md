  <name>Task 1: Author Agent Town map in Tiled editor</name>
  <read_first>
    - docs/tiled-authoring-guide.md (the complete guide produced in Plan 01)
  </read_first>
  <action>
    This task requires the user to design the map manually in the Tiled desktop editor. Claude cannot automate map authoring — this is visual, creative work that requires human judgment for building placement, interior decoration, and terrain design.

    The user follows docs/tiled-authoring-guide.md step by step:
    1. Create new 140x100 tile map at 32x32 pixel tiles
    2. Add all 16 CuteRPG tilesets
    3. Create all 14 layers in the exact order specified
    4. Design the town with central road layout (per D-02), commercial buildings along the main road, park area, homes on outskirts (per D-03, at least 8)
    5. Add reference-level interior detail to every building (per D-04)
    6. Create sector rectangles in the Sectors object layer with custom properties (per D-06, D-11)
    7. Create arena rectangles in the Arenas object layer using "sector:arena" naming (per D-06)
    8. Draw collision rectangles over all walls and non-walkable areas (per D-08)
    9. Place spawn point objects for all 8 agents inside their home sectors (per D-09)
    10. Set all metadata layers to visible: false
    11. Export as frontend/public/assets/tilemap/town.tmj
  </action>
  <how-to-verify>
    After exporting the TMJ file:
    1. Run the sync script to verify the TMJ is parseable:
       `python scripts/sync_map.py --tmj frontend/public/assets/tilemap/town.tmj`
    2. Check the output for any errors or warnings
    3. Run the validation script (created in Plan 03, or skip if not yet available):
       `python scripts/validate_map.py`
    4. Confirm the TMJ file exists: `ls -la frontend/public/assets/tilemap/town.tmj`
  </how-to-verify>
  <resume-signal>Type "map ready" when the TMJ file has been exported to frontend/public/assets/tilemap/town.tmj, or describe any issues encountered during map authoring.</resume-signal>
