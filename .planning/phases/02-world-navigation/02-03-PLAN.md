    ```python
    """Cross-plan integration tests: validate agent configs against generated town map."""
    import json
    from pathlib import Path

    import pytest
    from backend.simulation.world import Maze
    from backend.agents.loader import load_all_agents

    TOWN_JSON = Path(__file__).parent.parent / "backend" / "data" / "map" / "town.json"

    @pytest.fixture(scope="module")
    def maze():
        with open(TOWN_JSON) as f:
            return Maze(json.load(f))

    @pytest.fixture(scope="module")
    def agents():
        return load_all_agents()
    ```

    Add these tests:

    1. `test_all_agent_coords_within_map_bounds(agents, maze)`: For each agent, assert `0 <= agent.coord[0] < maze.width` and `0 <= agent.coord[1] < maze.height`.

    2. `test_all_agent_coords_on_walkable_tiles(agents, maze)`: For each agent, call `maze.tile_at(agent.coord)` and assert `tile.is_walkable` is True. If assertion fails, include the agent name and coord in the error message for debugging.

    3. `test_no_agent_spawns_on_border(agents, maze)`: For each agent, assert coord is not on row 0, row 99, col 0, or col 99 (border tiles are collision by design).

    4. `test_agent_home_sectors_exist_in_map(agents, maze)`: For each agent, extract the sector name from `agent.spatial.address["living_area"]` (index 1 of the address list, e.g. "home-alice" from `["agent-town", "home-alice", "bedroom"]`). Build the address key as `"agent-town:{sector}"` and assert it exists in `maze.address_tiles`.

    5. `test_agent_workplace_sectors_exist_in_map(agents, maze)`: For agents that have a "workplace" key in `agent.spatial.address`, extract the sector name (index 1) and assert it exists in `maze.address_tiles` as `"agent-town:{sector}"`.

    6. `test_all_agents_on_connected_graph(agents, maze)`: Pick the first agent's coord as a reference point. For every other agent, call `maze.find_path(agents[0].coord, agent.coord)` and assert the path is non-empty. This proves all agents spawn on the same connected walkable subgraph.

    7. `test_agent_spawn_coords_match_intended_sector(agents, maze)`: For each agent, check that the tile at their spawn coord has an address that includes the expected sector. For agents spawning at work (Alice at cafe, Bob at stock-exchange, etc.), verify the tile's address contains the workplace sector name. For agents spawning at home (Carla, Henry), verify the tile's address contains their home sector name. Use `maze.tile_at(agent.coord).address` to check. NOTE: If an agent's spawn coord is on an anonymous walkable tile (road) rather than inside a sector, this is a data quality issue that should be flagged -- log a warning but do not hard-fail (the coord validation tests above are the hard requirements).

    If any test fails because an agent's coord lands on a collision tile or out of bounds, the fix is to update the agent's JSON config to use a valid walkable coord within the intended sector. The test error message should include: agent name, coord, tile collision status, and the nearest walkable tiles in the intended sector (via `maze.resolve_destination`).
