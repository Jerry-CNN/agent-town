**Step 1: Create `backend/simulation/connection_manager.py`** (per D-04, D-05 research pattern):

```python
class ConnectionManager:
    def __init__(self):
        self.active_connections: list[WebSocket] = []

    async def connect(self, websocket: WebSocket):
        await websocket.accept()
        self.active_connections.append(websocket)

    def disconnect(self, websocket: WebSocket):
        if websocket in self.active_connections:
            self.active_connections.remove(websocket)

    async def broadcast(self, message: str) -> None:
        dead = []
        for ws in self.active_connections:
            try:
                await ws.send_text(message)
            except Exception:
                dead.append(ws)
        for ws in dead:
            self.active_connections.remove(ws)
```

**Step 3: Rewrite `backend/routers/ws.py`** (per D-05, D-08):

The endpoint must:
1. Accept the Request parameter to access `request.app.state.engine` and `request.app.state.connection_manager`
2. On connect: send snapshot FIRST via `engine.get_snapshot()`, wrapped in a WSMessage(type="snapshot"), THEN add to connection_manager (prevents delta race condition -- Pitfall 2)
3. Receive loop: parse WSMessage, handle "pause" -> engine.pause(), "resume" -> engine.resume(), "ping" -> send pong
4. On disconnect: remove from connection_manager
5. On invalid message: send WSMessage(type="error") as before

Import the Request from starlette for app.state access:
```python
from starlette.requests import Request

@router.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket, request: Request):
    engine = getattr(request.app.state, "engine", None)
    manager = getattr(request.app.state, "connection_manager", None)
```

Use getattr with None default so the endpoint does not crash if called before lifespan completes (graceful degradation). If engine is None, accept the connection and send an error message, then close.

When sending snapshot on connect, send it BEFORE calling `manager.connect()` -- this ensures the client receives the full state before any broadcast deltas (Pitfall 2 mitigation from RESEARCH.md).

Also broadcast a WSMessage(type="simulation_status", payload={"status": "paused"/"running"}) after pause/resume commands so all clients learn of the state change.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | ws.py WebSocket message parsing | mitigate | WSMessage.model_validate_json() with try/except; invalid messages return type="error" and do not crash the endpoint. Already established in Phase 1 ws.py stub. |
| T-04-05 | Denial of Service | ws.py pause/resume flood | accept | asyncio.Event set/clear is idempotent; rapid toggling has no meaningful cost. No rate limiting needed. |
| T-04-06 | Denial of Service | ConnectionManager.broadcast dead connections | mitigate | Dead connections caught in try/except per connection; removed from active list after broadcast loop completes. Does not mutate list during iteration. |
| T-04-07 | Denial of Service | SimulationEngine tick loop via CancelledError | mitigate | Tick loop catches asyncio.CancelledError only during shutdown via sim_task.cancel(). CancelledError is re-raised (not swallowed) so the task terminates cleanly. |
| T-04-08 | Information Disclosure | Snapshot exposes agent state | accept | Single-user v1; all agent state is public to the user. No PII in agent configs. |
</threat_model>
