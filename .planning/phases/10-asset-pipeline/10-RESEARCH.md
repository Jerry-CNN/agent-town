## Summary

Phase 10 is pure infrastructure: copy files, run a conversion script, and add one line of PixiJS initialization. No new npm packages are required. The phase has three concrete deliverables:

1. `frontend/public/assets/tilemap/` — 16 tileset PNGs copied from `~/projects/GenerativeAgentsCN/generative_agents/frontend/static/assets/village/tilemap/` (3MB total, individually small except interiors).
2. `frontend/public/assets/agents/` — 25 agent subdirectories (texture.png + portrait.png each, 110KB total), plus `sprite.json` in PixiJS spritesheet format generated by a conversion script.
3. `frontend/src/main.tsx` or `MapCanvas.tsx` — `TextureStyle.defaultOptions.scaleMode = 'nearest'` set before any `Assets.load()` call.

The critical knowledge for planning is: the reference `sprite.json` is Phaser-format (array frames, no animations dict, no meta.image) — it cannot be copied directly. A Python conversion script must produce a PixiJS-format JSON using the exact frame coordinates verified from inspection. All other assets are binary files with no format conversion required.

**Primary recommendation:** Structure as three sequential sub-plans: (1) tileset copy, (2) agent sprite copy + atlas conversion script, (3) scaleMode initialization. The first two can execute in parallel; the third is independent of both.

### Pattern 1: Phaser→PixiJS Atlas Conversion (Python script)

**What:** A one-time Python script reads the reference `sprite.json` (Phaser JSONArray format) and writes a new `sprite.json` in PixiJS `ISpritesheetData` format. No image manipulation — only JSON transformation.

**Reference format (confirmed by inspection):** [VERIFIED: direct inspection of reference sprite.json]
```json
{
  "frames": [
    { "filename": "down-walk.000", "frame": { "w": 32, "h": 32, "x": 0, "y": 0 }, "anchor": { "x": 0.5, "y": 0.5 } },
    ...
  ],
  "meta": { "description": "Atlas generado con Atlas Packer Gamma V2", ... }
}
```
- `frames` is an **array** (PixiJS requires **object/dict**)
- No `animations` section (PixiJS AnimatedSprite requires this)
- No `meta.image` field (PixiJS loader requires this to locate the PNG)

**Python conversion logic (no third-party deps):**
```python
import json, shutil
from pathlib import Path

REF_JSON = Path("~/projects/GenerativeAgentsCN/.../agents/sprite.json").expanduser()
OUT_JSON = Path("frontend/public/assets/agents/sprite.json")

with open(REF_JSON) as f:
    src = json.load(f)

frames_dict = {}
for entry in src["frames"]:
    name = entry["filename"]
    fr = entry["frame"]
    frames_dict[name] = {
        "frame": {"x": fr["x"], "y": fr["y"], "w": fr["w"], "h": fr["h"]},
        "sourceSize": {"w": fr["w"], "h": fr["h"]},
    }

output = {
    "frames": frames_dict,
    "animations": {
        "down-walk":  ["down-walk.000", "down-walk.001", "down-walk.002", "down-walk.003"],
        "left-walk":  ["left-walk.000", "left-walk.001", "left-walk.002", "left-walk.003"],
        "right-walk": ["right-walk.000", "right-walk.001", "right-walk.002", "right-walk.003"],
        "up-walk":    ["up-walk.000", "up-walk.001", "up-walk.002", "up-walk.003"],
    },
    "meta": {
        "image": "texture.png",
        "format": "RGBA8888",
        "size": {"w": 96, "h": 128},
        "scale": 1,
    },
}

OUT_JSON.parent.mkdir(parents=True, exist_ok=True)
with open(OUT_JSON, "w") as f:
    json.dump(output, f, indent=2)
print(f"Written: {OUT_JSON}")
```

## Common Pitfalls

### Pitfall 1: Copying sprite.json from Reference Silently Produces Invisible Sprites

**What goes wrong:** Phaser JSONArray format is structurally incompatible with PixiJS `ISpritesheetData`. PixiJS does not throw on load — it returns an empty spritesheet. Sprites created from it have zero dimensions and render invisible. No console error.

**Root cause:** Both formats are called "texture atlas JSON" but use different schemas. Phaser uses array frames; PixiJS uses keyed object frames + mandatory `animations` section + `meta.image`.

**Prevention:** The Python conversion script produces the correct format. Never use the reference `sprite.json` directly. Test by loading in a minimal HTML test page (not inside React) and checking `sheet.textures['down']` is non-null after load. [VERIFIED: PITFALLS.md Pitfall 1]

### Pitfall 2: TextureStyle.defaultOptions.scaleMode Set Too Late

**What goes wrong:** If `scaleMode = 'nearest'` is set inside a `useEffect` or after any `Assets.load()` call, previously loaded textures retain linear (blurry) filtering permanently. The setting only affects textures created after it is set.

**Prevention:** Set at module level in `MapCanvas.tsx` (outside any function) or in `main.tsx` before any React rendering begins. [VERIFIED: PITFALLS.md Pitfall 5, STATE.md key decisions]

**Warning signs:** Tiles look blurry at zoom. Individual pixels appear as soft gradients.

### Pitfall 3: Tileset Image Paths Break When Assets Are Reorganized

**What goes wrong:** pixi-tiledmap resolves tileset image paths relative to the map JSON's URL. If the tileset PNGs are not at the expected relative path from the map JSON, they 404 silently, rendering as blank tiles.

**Prevention:** Keep tileset PNGs co-located in the same directory as the future `town.tmj` (`frontend/public/assets/tilemap/`). Phase 11 (Tiled map authoring) must keep the Tiled project file inside `public/assets/tilemap/` so relative paths stay valid. This phase must establish the correct directory structure that Phase 11 depends on. [VERIFIED: PITFALLS.md Pitfall 2]

### Pitfall 4: Missing `frontend/public/` Directory

**What goes wrong:** `frontend/public/` does not exist (confirmed). Vite expects it to exist as the static file root. If it is absent, Vite dev server will not serve any assets from `/assets/`.

**Prevention:** First task of this phase creates `frontend/public/assets/tilemap/` and `frontend/public/assets/agents/` directories. [VERIFIED: ls check confirms public/ is absent]

### Pitfall 5: Agent Directories Kept in Chinese — Causes URL Encoding Issues

**What goes wrong:** If agent sprite directories use Chinese character names (e.g., `梅/texture.png`), URLs like `/assets/agents/梅/texture.png` require percent-encoding (`%E6%A2%85`). While browsers handle this, the `data.imageFilename` override in `Assets.add()` must exactly match the URL-encoded path or use the decoded form — inconsistency causes 404s.

**Prevention:** Rename the 8 active agent directories to English names (alice, bob, etc.) during the copy step. Remaining 17 can use index names (`agent_09` through `agent_25`) to avoid encoding issues entirely. [ASSUMED — based on URL encoding behavior; not a confirmed PixiJS-specific issue]
