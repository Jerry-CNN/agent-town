```python
import math
from backend.simulation.world import Maze
from backend.schemas import PerceptionResult

def perceive(
    agent_coord: tuple[int, int],
    agent_name: str,
    maze: Maze,
    all_agents: dict[str, dict],
    radius: int = 5,
) -> PerceptionResult:
```

Implementation per D-06 (tile-based vision radius) and D-07 (perceive agents, events, location):
- `all_agents` parameter is a dict mapping agent_name -> dict with keys "coord" (tuple[int,int]) and "current_activity" (str). This decouples from any specific AgentState class not yet defined.
- Iterate over all tiles within a square bounding box from `(x-radius, y-radius)` to `(x+radius, y+radius)`.
- For each candidate tile, compute Euclidean distance: `math.sqrt(dx**2 + dy**2)`. Skip if distance > radius.
- Use `maze.tile_at((tx, ty))` with try/except IndexError to handle edge-of-map coordinates.
- Collect tile `_events` dict entries as `{"distance": float, "event": event_data}`.
- Collect nearby agents by checking `all_agents` entries whose "coord" matches the scanned tile. Skip self (agent_name). Build `{"name": str, "activity": str, "distance": float}`.
- Sort nearby_events and nearby_agents by distance ascending (closest first).
- Cap nearby_events at 10 entries (per reference att_bandwidth limit).
- Get current tile location context: `maze.tile_at(agent_coord).get_address(as_list=False)` for the location string.
- Return `PerceptionResult(nearby_events=events[:10], nearby_agents=agents, location=location_str)`.

CRITICAL: This function is synchronous (not async). Perception is a pure Python computation with ZERO LLM calls. It reads the tile grid directly. This is intentional per 03-RESEARCH.md Anti-Pattern 1.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM response -> DailySchedule | Untrusted LLM output parsed into DailySchedule (activity list, wake_hour) |
| LLM response -> SubTask list | Untrusted LLM output parsed into SubTask list (time ranges) |
| Tile grid -> perception | Tile data is server-controlled, trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | DailySchedule LLM response | mitigate | Pydantic Field(ge=4, le=11) on wake_hour; Field(min_length=3) on activities; instructor retries on validation failure |
| T-03-06 | Tampering | SubTask time ranges | mitigate | Pydantic Field(ge=0, lt=1440) on start_minute; Field(ge=5, le=60) on duration_minutes; invalid values rejected |
| T-03-07 | Denial of Service | Schedule generation LLM calls | accept | Only 2 LLM calls per schedule generation; max_retries=3 via complete_structured; bounded cost |
| T-03-08 | Information Disclosure | Perception radius | accept | Perception reads server-side tile data only; no user-supplied input crosses this boundary |
</threat_model>
