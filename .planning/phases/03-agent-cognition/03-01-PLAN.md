```python
async def retrieve_memories(
    simulation_id: str,
    agent_id: str,
    query: str,
    top_k: int = 10,
    recency_decay: float = 0.995,
    recency_weight: float = 0.5,
    relevance_weight: float = 3.0,
    importance_weight: float = 2.0,
) -> list[Memory]:
```

Implementation details:
- Import `get_collection` from `backend.agents.memory.store`
- Over-fetch from ChromaDB: `n_results = min(50, top_k * 5)` to ensure enough candidates for re-ranking
- Query with `where={"agent_id": agent_id}` -- MANDATORY per D-01 (Pitfall 2: cross-agent contamination)
- Wrap `col.query()` with `asyncio.to_thread()` -- MANDATORY per Pitfall 1 (sync ChromaDB blocking event loop)
- Composite score formula (per D-02):
  - `recency = recency_decay ** hours_since_last_access` where `hours_since_last_access = (now - meta["last_access"]) / 3600`
  - `relevance = 1.0 - distance` (ChromaDB cosine distance: 0=identical, 1=orthogonal, so 1-dist gives similarity)
  - `importance = meta["importance"] / 10.0` (normalize to 0-1 range)
  - `score = recency * recency_weight + relevance * relevance_weight + importance * importance_weight`
- Sort by score descending, return top_k as list of `Memory` Pydantic models
- Handle empty results gracefully (return [])
- Update `last_access` timestamp on retrieved memories (write-back via `col.update()` in `asyncio.to_thread`)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM response -> importance scoring | Untrusted LLM output parsed into ImportanceScore (1-10 integer) |
| ChromaDB metadata -> retrieval ranking | Metadata values (importance, timestamps) used in scoring formula |
| Agent ID scoping | Queries must filter by agent_id to prevent cross-agent memory contamination |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | ImportanceScore LLM response | mitigate | Pydantic Field(ge=1, le=10) enforces range; instructor retries on validation failure; hardcoded fallback=5 on exhausted retries |
| T-03-02 | Information Disclosure | Memory retrieval | mitigate | Enforce where={"agent_id": agent_id} on every ChromaDB query in retrieve_memories function signature -- impossible to call without it |
| T-03-03 | Denial of Service | Importance scoring LLM calls | mitigate | Skip LLM call for idle events (hardcode importance=1); max_retries=3 on complete_structured; failsafe return 5 |
| T-03-04 | Elevation of Privilege | ChromaDB collection naming | accept | Collection names use f"sim_{simulation_id}" -- simulation_id is server-generated, not user-supplied in v1 |
</threat_model>
