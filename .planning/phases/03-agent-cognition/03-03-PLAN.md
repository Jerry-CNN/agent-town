```python
import time
from backend.gateway import complete_structured
from backend.schemas import (
    ConversationDecision, ConversationTurn, ScheduleRevision,
    ScheduleEntry, AgentScratch, ImportanceScore
)
from backend.agents.memory.store import add_memory, score_importance
from backend.agents.memory.retrieval import retrieve_memories

MAX_TURNS = 4  # D-12: hard cap at 4 turns
COOLDOWN_SECONDS = 60  # Claude's discretion: 60 real-time seconds between same-pair conversations

# In-memory cooldown tracker: frozenset({agent_a, agent_b}) -> last_conversation_timestamp
_conversation_cooldowns: dict[frozenset, float] = {}

def _pair_key(agent_a: str, agent_b: str) -> frozenset:
    return frozenset({agent_a, agent_b})

def check_cooldown(agent_a: str, agent_b: str) -> bool:
    """Returns True if enough time has passed since last conversation."""
    key = _pair_key(agent_a, agent_b)
    last_time = _conversation_cooldowns.get(key, 0)
    return (time.time() - last_time) >= COOLDOWN_SECONDS

def _record_conversation(agent_a: str, agent_b: str) -> None:
    _conversation_cooldowns[_pair_key(agent_a, agent_b)] = time.time()

async def attempt_conversation(
    simulation_id: str,
    agent_name: str,
    agent_scratch: AgentScratch,
    other_name: str,
    other_activity: str,
    agent_current_activity: str,
    location: str,
) -> bool:
```

Implementation per D-12 (2-4 turns, capped at 4) and D-13 (schedule revision after):
- Initialize `conversation_log: list[dict] = []` to track turns.
- **Turn loop:** `for turn in range(MAX_TURNS)`:

  - If `result.end_conversation` is True and `turn >= 1`: break.

  - Score importance via `score_importance()` for each agent.
  - Call `add_memory(simulation_id, agent_a_name, summary_a, "conversation", importance_a)`.
  - Call `add_memory(simulation_id, agent_b_name, summary_b, "conversation", importance_b)`.
- **Schedule revision (D-10, D-13):**

  - Return `{"turns": conversation_log, "revised_schedule_a": revision_a.revised_entries, "revised_schedule_b": revision_b.revised_entries, "summary": summary_text}`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM response -> ConversationDecision | Untrusted LLM output determines whether agents converse |
| LLM response -> ConversationTurn | Untrusted LLM generates dialogue content |
| LLM response -> ScheduleRevision | Untrusted LLM modifies agent schedule entries |
| LLM response -> AgentAction | Untrusted LLM chooses agent destination and activity |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Denial of Service | Conversation loop | mitigate | Hard cap MAX_TURNS=4 via `for range(MAX_TURNS)` -- never `while True`; cooldown prevents re-initiation within 60 seconds |
| T-03-10 | Tampering | ConversationTurn.text | accept | Free-text dialogue is expected; content is stored as memory summary, not executed. No code injection vector. |
| T-03-11 | Tampering | ScheduleRevision.revised_entries | mitigate | Each revised ScheduleEntry validated by Pydantic Field constraints (start_minute 0-1439, duration 15-120 min); invalid entries rejected |
| T-03-12 | Denial of Service | Memory storage after conversation | mitigate | Limited to 2 memories per conversation (one per agent); importance scoring has failsafe return 5 on LLM failure |
| T-03-13 | Tampering | AgentAction.destination | accept | Destination string is resolved via Maze.resolve_destination() which returns None for unknown sectors; caller handles None case |
| T-03-14 | Information Disclosure | Memory retrieval in decide_action | mitigate | retrieve_memories enforces agent_id filter (from Plan 01); agents cannot access other agents' memories |
</threat_model>
