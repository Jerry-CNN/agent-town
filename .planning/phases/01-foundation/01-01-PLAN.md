**backend/gateway.py** — The core async LLM gateway. Key implementation details:

```python
import asyncio
import time
import logging
from typing import TypeVar, Type
import instructor
import litellm
from backend.schemas import AgentAction, ProviderConfig
from backend import config as cfg

logger = logging.getLogger(__name__)

T = TypeVar("T")

# Disable LiteLLM verbose logging — it's very noisy
litellm.suppress_debug_info = True

FALLBACK_AGENT_ACTION = AgentAction(
    destination="idle",
    activity="waiting",
    reasoning="LLM parse failed"
)

# instructor wraps litellm.acompletion — same interface across providers
_client = instructor.from_litellm(litellm.acompletion)


def _resolve_model(provider_config: ProviderConfig) -> tuple[str, dict]:
    """Returns (model_string, litellm_kwargs) for the given provider config."""
    if provider_config.provider == "ollama":
        model = provider_config.model or cfg.OLLAMA_DEFAULT_MODEL
        return model, {"api_base": cfg.OLLAMA_BASE_URL}
    else:  # openrouter
        model = provider_config.model or cfg.OPENROUTER_DEFAULT_MODEL
        return model, {"api_key": provider_config.api_key}


async def complete_structured(
    messages: list[dict],
    response_model: Type[T],
    provider_config: ProviderConfig | None = None,
    max_retries: int = 3,
) -> T:
    """
    Make an async LLM call and return a validated Pydantic model.
    Uses instructor for automatic retry on validation failure.
    Returns a fallback value if all retries fail — never raises.
    """
    if provider_config is None:
        provider_config = ProviderConfig(
            provider=cfg.state.provider,
            api_key=cfg.state.api_key,
            model=cfg.state.model,
        )

    model_str, litellm_kwargs = _resolve_model(provider_config)

    try:
        result = await _client.chat.completions.create(
            model=model_str,
            messages=messages,
            response_model=response_model,
            max_retries=max_retries,
            **litellm_kwargs,
        )
        return result
    except Exception as exc:
        logger.warning("LLM call failed after %d retries: %s", max_retries, exc)
        # Return type-safe fallback if response_model is AgentAction
        if response_model is AgentAction:
            return FALLBACK_AGENT_ACTION  # type: ignore[return-value]
        raise
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → POST /api/config | User-supplied OpenRouter API key crosses into backend state |
| Backend → LiteLLM → OpenRouter | API key leaves the process boundary to an external cloud provider |
| Backend → Ollama (localhost) | Localhost-only; not an internet boundary in dev |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | POST /api/config | accept | Single-user local app; no auth in v1. Architecture note: API key stored in backend process memory (AppState), not logged. |
| T-01-02 | Information Disclosure | backend/config.py AppState | mitigate | Never log `api_key` value. In gateway.py logger.warning must not include the api_key in the message string. Use `api_key[:8]+"..."` if key must be referenced in logs. |
| T-01-03 | Denial of Service | LLM gateway retry loop | mitigate | `max_retries=3` cap in `complete_structured`. instructor retries are bounded. Ollama health check uses 3-second httpx timeout to prevent startup hang. |
| T-01-04 | Tampering | ProviderConfig Pydantic validation | mitigate | Pydantic v2 validator rejects openrouter config without api_key (422). Model string is passed directly to LiteLLM — validate that model string matches expected pattern `^(ollama_chat/|openrouter/).+` before forwarding. |
</threat_model>
