<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /api/config | OpenRouter API key crosses from browser to backend process memory |
| localStorage → Zustand | Persisted provider config (including API key) re-hydrated on page load |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | localStorage `agenttown_provider` | accept | Single-user local app. API key stored in localStorage is a known UX tradeoff — acceptable for a developer-facing local tool. No server-side storage. Key is NOT logged to console anywhere. |
| T-03-02 | Spoofing | ProviderSetup fetch to POST /api/config | accept | Single-user localhost. No CSRF concern since no session cookies or cross-origin requests in v1. |
| T-03-03 | Tampering | localStorage config re-hydration | mitigate | Wrap `JSON.parse(localStorage.getItem("agenttown_provider"))` in try/catch. If parse fails or schema is invalid (missing `provider` field), treat as unconfigured and show modal. Never trust localStorage data without validation. |
| T-03-04 | Information Disclosure | OllamaStatusBanner | accept | Banner shows generic "Ollama not running" message — no internal details exposed to UI. API key never mentioned in banner text. |
</threat_model>
