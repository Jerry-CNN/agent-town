Zustand store shape (all future phases extend this):
```typescript
interface SimulationState {
  connected: boolean
  ollamaAvailable: boolean | null  // null = unknown (startup polling)
  feedEntries: FeedEntry[]
  setConnected: (v: boolean) => void
  setOllamaStatus: (available: boolean) => void
  addFeedEntry: (entry: FeedEntry) => void
}
// Max 200 feed entries: feedEntries: [entry, ...s.feedEntries].slice(0, 200)
```

FeedEntry interface (stable contract — do NOT change field names):
```typescript
interface FeedEntry {
  id: string           // crypto.randomUUID()
  timestamp: number    // Date.now()
  message: string
  type: 'agent_action' | 'agent_thought' | 'system' | 'error'
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser WebSocket → FastAPI | Client sends JSON messages; handled by ws.ts |
| Frontend → /api/* | REST calls proxied by Vite to FastAPI; no auth in Phase 1 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ws.ts WebSocket message handler | mitigate | Parse incoming JSON with try/catch; ignore malformed messages. Never eval/execute message content. |
| T-02-02 | Information Disclosure | sessionStorage session_id | accept | Session ID is a UUID with no user PII. Cleared on tab close by default. |
| T-02-03 | Denial of Service | Zustand feed capped at 200 entries | mitigate | `slice(0, 200)` already implemented in addFeedEntry. Prevents memory growth from high-volume WS messages. |
| T-02-04 | Spoofing | No auth on WebSocket /ws/{session_id} | accept | Local-only in Phase 1. Auth will be added if multi-user support is required in future phases. |
</threat_model>
