<objective>
Add the backend event injection pipeline: extend WSMessage schema with "inject_event" type, add SimulationEngine.inject_event() method that stores high-importance memories, and wire the ws.py handler to dispatch incoming inject_event messages to the engine and broadcast confirmation events to the activity feed.

Purpose: This is the backend plumbing that makes user-injected events flow into agent memory streams. Without this, the frontend UI (Plan 02) has nothing to send to. EVT-04 (gossip spreading) requires no new code -- it works because inject_event stores memories at importance=8, which the existing composite retrieval scorer surfaces in future agent conversations.

Output: Modified backend/schemas.py, backend/simulation/engine.py, backend/routers/ws.py, new tests/test_event_injection.py
