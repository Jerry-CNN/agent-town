<objective>
Add concurrency control and adaptive tick timing to the LLM gateway and simulation engine.

Purpose: Prevent rate-limit errors and queue pileup by bounding concurrent LLM calls with asyncio.Semaphore(8) (LLM-04). Make the simulation tick interval self-adjusting based on actual LLM response latency (LLM-02), so fast providers (OpenRouter) get ~10s ticks and slow providers (Ollama) get proportionally longer ticks automatically.

Output: Modified gateway.py with semaphore + latency tracking, modified engine.py with adaptive tick interval, updated Agent class with per-sector gating fields, passing tests.
