# Phase 09.1 Plan 01: Backend Runtime Wiring Summary

**One-liner:** Wired Event lifecycle registry and Agent OOP wrappers into SimulationEngine, closing 4 v1.1 audit gaps (EVTS-01/02/03, ARCH-02) with 6 new integration tests.

## What Was Built

Four gap-closure wiring changes to `backend/simulation/engine.py`:

1. **EVTS-01 — Event object registry:** `inject_event()` now creates an `Event` Pydantic object (status transitions created→active via `event.tick()`) and stores it in a new `self._active_events: dict[str, Event]` field. ChromaDB `add_memory()` call is preserved — Event lifecycle tracking is additive, not a replacement.

2. **EVTS-02 — heard_by propagation:** After every `agent.perceive()` call in `_agent_step()`, whisper events in `_active_events` get the agent's name appended to `ev.heard_by` if not already present. Broadcast events skip `heard_by` per D-09.

3. **EVTS-03 — event expiry:** At the end of each `_tick_loop()` iteration (after `_tick_count` increments), all active events have `ev.tick()` called and are deleted from `_active_events` if `ev.is_expired()` returns True.

4. **ARCH-02 — Agent wrapper delegation:** `_agent_step()` now calls `agent.perceive(maze=self.maze, all_agents=...)` and `agent.decide(simulation_id=..., perception=..., ...)` instead of the module-level `perceive()` and `decide_action()` functions. The `agent.decide()` wrapper reads `agent_scratch`, `agent_spatial`, `current_activity`, and `current_schedule` from `self` fields internally.

## Decisions Made

- `_active_events` dict is keyed by `str(uuid.uuid4())` — stable IDs that don't embed event content, avoiding accidental key collisions on duplicate event texts.
- `heard_by` is updated inside `_agent_step()` per-tick, not inside `inject_event()`. This ensures `heard_by` reflects actual runtime simulation propagation (the agent actually "perceived" the event in a given tick), not just injection targeting.
- `add_memory()` call in `inject_event()` is preserved — ChromaDB storage and Event lifecycle tracking serve different purposes (retrieval vs. expiry/propagation tracking).
- Module-level `perceive` import removed from engine.py; `decide_action` import replaced with `_extract_known_locations` only (still needed for `open_locs` computation at line 400).
