# Phase 9.2: Visual Text Restoration - Research

**Researched:** 2026-04-11
**Domain:** PixiJS v8 / @pixi/react v8 — imperative text rendering, WCAG contrast
**Confidence:** HIGH

---

## Summary

Phase 9.2 restores activity text to agent sprites and verifies name label contrast. The
work is entirely confined to `frontend/src/components/AgentSprite.tsx`. No backend changes
are required.

Commit `686ddf7` simplified agent sprites by removing the activity text (`pixiText` node)
and both pill background `pixiGraphics` nodes, leaving only the circle, initial letter, and
name label. The removal was intentional UX simplification but left VIS-02 only half
satisfied: names are visible but activities are not shown without clicking.

Contrast analysis (run against all eight sector colors in TileMap.tsx) shows the current
dark navy text (`0x1a1a2e`) passes WCAG AA (>= 4.5:1) on every sector background by a
wide margin. The original audit concern about contrast is a false alarm. The real gap is
the missing activity text.

The previous implementation (before `686ddf7`) used a pill background with `alpha: 0.7`
and white text, and updated the activity via imperative `ref.current.text` mutation inside
`useTick`. This pattern is confirmed correct for PixiJS v8 + @pixi/react v8 and must be
restored.

**Primary recommendation:** Restore activity text exactly as it existed before `686ddf7`,
adapting the font size and pill dimensions to match the enlarged sprite (RADIUS now 18, up
from 12). No contrast fix is needed for the name label — it already passes.

### Pattern 1: Imperative PixiJS Text Update in useTick
**What:** Activity text is stored as a `useRef<PixiText | null>(null)`. Inside `useTick`,
after lerping position, the ref's `.text` property is mutated directly. This bypasses
React's render cycle entirely — no `setState`, no re-render.

**When to use:** Any value that changes every few seconds (agent activity) and must update
on the map without causing a React re-render. The project already uses this for position
(`currentPosRef`, `containerRef`).

**Example (from commit before 686ddf7 — verified via `git show 686ddf7`):**
```typescript
// Source: git show 686ddf7 (pre-removal code)
const activityTextRef = useRef<PixiText | null>(null);

// In useTick:
if (activityTextRef.current) {
  const newActivity = truncateActivity(agent.activity ?? "");
  if (activityTextRef.current.text !== newActivity) {
    activityTextRef.current.text = newActivity;
  }
}

// In JSX:
<pixiText
  ref={activityTextRef}
  text={initialActivity}
  x={0}
  y={-19}
  anchor={{ x: 0.5, y: 1 }}
  style={{
    fontFamily: "system-ui, sans-serif",
    fontSize: 10,
    fill: 0xffffff,
  }}
/>
```
The guard `if (activityTextRef.current.text !== newActivity)` avoids unnecessary PixiJS
re-layout on frames when activity hasn't changed.

### Pattern 2: Semi-transparent Pill Background via pixiGraphics
**What:** A `pixiGraphics` node drawn before the text provides a rounded-rect backdrop for
readability. The pill uses `alpha: 0.7` so the sector color shows through without
overwhelming the label. The draw callback has an empty `useCallback` deps array because the
pill geometry is constant.

**When to use:** Any text overlaid on a colored map background. Required here because
activity text sits inside sector zones of varying colors.

**Example (from commit before 686ddf7):**
```typescript
// Source: git show 686ddf7 (pre-removal code)
const drawActivityPill = useCallback((g: PixiGraphics) => {
  g.clear();
  g.setFillStyle({ color: 0x111111, alpha: 0.7 });
  g.roundRect(-55, -26, 110, 14, 4);
  g.fill();
}, []);
```

**Sizing note for Phase 9.2:** RADIUS was increased from 12 to 18 in `686ddf7`. The pill
dimensions and text y-offsets must be scaled to match the larger circle. Recommended
adjustments (to be confirmed visually):
- Activity pill: wider to match text content (~`-60, -30, 120, 16, 4`)
- Activity text `y`: ~`-24` (above top of 18px circle)
- Name pill: wider to match 20px bold name (`-55, 20, 110, 20, 5`)

### Pattern 3: Truncation for Map Labels
**What:** Activity strings from the LLM can be long. The previous code truncated to 18
chars with `…`. This prevents label overflow beyond the pill.

**When to use:** Always for activity text on map sprites.

```typescript
// Source: git show 686ddf7 (pre-removal code)
const MAX_ACTIVITY_LEN = 18;
function truncateActivity(text: string): string {
  if (text.length <= MAX_ACTIVITY_LEN) return text;
  return text.slice(0, MAX_ACTIVITY_LEN - 1) + "…";
}
```

### Anti-Patterns to Avoid
- **React state for activity text:** Using `useState` for activity text triggers a full
  React re-render on every `agent_update` WebSocket message. With 8 agents updating every
  few seconds this causes visible jitter. Use imperative ref mutation in `useTick`.
- **Rebuilding `pixiGraphics` draw callbacks with changing deps:** The pill draw callback
  must have stable/empty deps or it redraws every frame, degrading performance.
- **Removing the text-equality guard:** Mutating `.text` unconditionally in `useTick` calls
  PixiJS text layout even when text hasn't changed. Always guard with `!==`.

## Common Pitfalls

### Pitfall 1: Pill Geometry Mismatch After Radius Increase
**What goes wrong:** The pill drawn with pre-`686ddf7` dimensions (`-55, -26, 110, 14`)
will clip or misalign against the larger 18px circle (was 12px).
**Why it happens:** The commit that removed activity text also increased RADIUS from 12 to
18. The pill offsets were calculated for the smaller radius.
**How to avoid:** Re-measure pill geometry visually. Start with `y` offset for activity
pill at approximately `-RADIUS * 2` and adjust.
**Warning signs:** Activity text appears inside the circle or too close to the name label.

### Pitfall 2: Text Ref Not Attached on Mount
**What goes wrong:** `activityTextRef.current` is null in the first few `useTick` frames
if the JSX node hasn't mounted yet.
**Why it happens:** PixiJS refs populate after first paint.
**How to avoid:** Always guard with `if (activityTextRef.current)` before mutating — this
is already in the pattern. Do not add `useEffect` dependency on the ref.

### Pitfall 3: Import Regression
**What goes wrong:** `Text as PixiText` was removed from the pixi.js import in `686ddf7`.
Restoring activity text requires restoring this import.
**Why it happens:** The simplification commit removed the import when it removed the ref.
**How to avoid:** First step of the plan: restore
`import { ..., Text as PixiText, ... } from "pixi.js";`

// Source: git show 686ddf7 (verified pre-removal pattern)

  return text.slice(0, MAX_ACTIVITY_LEN - 1) + "…";

  ref={activityTextRef}

## Open Questions

1. **Name pill: restore or not?**
   - What we know: Contrast passes without it (dark navy on all backgrounds).
   - What's unclear: Whether bare text looks good against the sector colors visually, even
     if it's technically contrast-compliant.
   - Recommendation: Do not restore the name pill. Dark navy text passes WCAG AA cleanly
     and is simpler. If a human reviewer disagrees during verification, it's a one-line add.

2. **Activity truncation length: 18 chars still correct?**
   - What we know: 18 was chosen for RADIUS=12. With RADIUS=18 and a wider pill, could fit
     20-22 chars.
   - What's unclear: Exact readable width at 0.45x zoom.
   - Recommendation: Keep 18 chars for the first implementation; increase if visual review
     shows text is truncating too aggressively.
