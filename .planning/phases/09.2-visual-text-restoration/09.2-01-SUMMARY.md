# Phase 09.2 Plan 01: Restore Activity Text on Agent Sprites Summary

**One-liner:** Restored activity text with semi-transparent pill background on agent sprites using imperative PixiText ref mutation in useTick, adapted for RADIUS=18.

## What Was Built

Commit `686ddf7` had removed activity text and pill backgrounds from AgentSprite.tsx, leaving agents showing only name and initial without any visible activity. This plan closes the VIS-02 gap by restoring the proven pre-removal pattern, adjusted for the enlarged RADIUS=18 sprite.

Changes to `frontend/src/components/AgentSprite.tsx`:
- Restored `Text as PixiText` import from pixi.js
- Added `truncateActivity()` helper (MAX_ACTIVITY_LEN = 18 chars, ellipsis suffix)
- Added `activityTextRef = useRef<PixiText | null>(null)` alongside existing containerRef
- Added `initialActivity` read from store (stable on first render)
- Extended `useTick` callback: after lerp position, imperatively updates `activityTextRef.current.text` with text-equality guard (only updates on actual change — mitigates T-9.2-02 DoS via unnecessary PixiJS text re-layout)
- Added `drawActivityPill` useCallback: semi-transparent dark pill (0x111111, alpha 0.7), roundRect(-60, -30, 120, 16, 4)
- Added pill + pixiText JSX nodes above the circle: y=-24, anchor={0.5,1}, fontSize=11, fill=0xffffff
