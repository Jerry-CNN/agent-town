# Phase 09.2: Code Review Report

**Reviewed:** 2026-04-11
**Depth:** standard
**Files Reviewed:** 1
**Status:** issues_found

## Summary

`AgentSprite.tsx` is a well-structured PixiJS v8 component with correct separation of the
imperative tick loop from React rendering. The core architecture (useTick for lerp, refs for
display objects, getState() for non-reactive per-frame reads) follows the project's performance
contract correctly.

Three warnings were found:

1. A stale-data bug where `agentName` and `initialLetter` are frozen from the first render and
   never update if the agent enters the store after mount.
2. The initial letter centering uses hard-coded offsets that do not properly center glyphs —
   the anchor is wrong.
3. The activity pill background geometry does not fully contain the activity text — the top of
   the text overflows the pill rect by approximately 5px.

Two info items cover the name label font size (20px, larger than the circle) and magic numbers
in the pill draw callback.

## Warnings

### WR-01: Stale agent name and initial letter on late mount

**File:** `frontend/src/components/AgentSprite.tsx:103-106`

**Issue:** `agentName`, `initialLetter`, and `initialActivity` are computed by calling
`useSimulationStore.getState()` directly at render time (not inside a reactive selector
or `useEffect`). They are evaluated once at mount and never updated. If the component
mounts before `updateAgentsFromSnapshot` fires — which can happen on initial page load or
reconnection — `initialAgent` is `undefined`, `agentName` falls back to `agentId`, and the
name label permanently displays the raw agent ID string instead of the actual name. The
`useEffect` on line 71 correctly initializes `currentPosRef` when `agentId` changes, but
there is no equivalent mechanism to refresh `agentName` when the store later populates.

**Fix:** Derive the name reactively using the Zustand hook with a stable selector to avoid
re-renders on activity-only changes:

```tsx
// Replace lines 103-106 with:
const agentName = useSimulationStore(
  useCallback((s) => s.agents[agentId]?.name ?? agentId, [agentId])
);
const initialLetter = agentName.charAt(0).toUpperCase();

// initialActivity stays imperative (read in useTick), remove it from render scope:
// const initialActivity = ""; // useTick handles updates via activityTextRef
```

The name and initial letter are essentially static once assigned (agents don't rename
themselves), so the reactive subscription only fires once in practice — the cost is
negligible.

### WR-02: Initial letter not centered on circle — wrong anchor and hard-coded offsets

**File:** `frontend/src/components/AgentSprite.tsx:163-173`

**Issue:** The `pixiText` for the initial letter uses `x={-6} y={-9}` with no explicit
`anchor` prop. PixiJS v8 defaults to `anchor = {x: 0, y: 0}` (top-left origin), so the
actual centering depends on the rendered glyph width and height, which vary per character.
`RADIUS = 18` (diameter 36px) and `fontSize = 18`. An "M" or "W" will be visibly right of
center; an "I" will be visibly left. The offsets `-6` and `-9` are approximations that work
for mid-width characters only.

**Fix:** Set `anchor={{ x: 0.5, y: 0.5 }}` and position at the circle center:

```tsx
<pixiText
  text={initialLetter}
  x={0}
  y={0}
  anchor={{ x: 0.5, y: 0.5 }}
  style={{
    fontFamily: "system-ui, sans-serif",
    fontSize: 18,
    fontWeight: "bold",
    fill: 0xffffff,
  }}
/>
```

This lets PixiJS center the glyph regardless of its measured width.

### WR-03: Activity text overflows pill background by ~5px at top

**File:** `frontend/src/components/AgentSprite.tsx:82-157`

**Issue:** The activity pill `roundRect` is drawn at `(-60, -30, 120, 16, 4)`, which means
it occupies y from `-30` to `-14`. The `pixiText` for the activity is positioned at
`y={-24}` with `anchor={{ x: 0.5, y: 1 }}` (bottom of text at y=-24, text grows upward).
With `fontSize = 11`, the glyph ascends approximately 11px above the baseline, putting the
top of the text at approximately `y = -35`. The pill top is at y=-30. The text top exceeds
the pill background by roughly 5px, so the top portion of letters renders on the transparent
background rather than inside the pill.

**Fix:** Either shift the pill rectangle up to accommodate the full text height, or adjust
the text anchor/position:

```tsx
// Option A: move pill up by 6px (rect from -36 to -20) and shift text to match
const drawActivityPill = useCallback((g: PixiGraphics) => {
  g.clear();
  g.setFillStyle({ color: 0x111111, alpha: 0.7 });
  g.roundRect(-60, -36, 120, 18, 4); // taller rect, shifted up
  g.fill();
}, []);

// Corresponding text change: y={-22} with anchor y=0.5 (vertically centered in pill)
<pixiText
  ref={activityTextRef}
  text={initialActivity}
  x={0}
  y={-27}
  anchor={{ x: 0.5, y: 0.5 }}
  style={{ fontFamily: "system-ui, sans-serif", fontSize: 11, fill: 0xffffff }}
/>

// Option B (minimal): change text anchor to y=0 and position at y=-30 so text starts
// inside the pill and stays within its 16px height
<pixiText
  ref={activityTextRef}
  text={initialActivity}
  x={0}
  y={-30}
  anchor={{ x: 0.5, y: 0 }}
  style={{ fontFamily: "system-ui, sans-serif", fontSize: 11, fill: 0xffffff }}
/>
```

## Info

### IN-01: Name label font size (20px) exceeds circle diameter (36px) and will cause label collisions

**File:** `frontend/src/components/AgentSprite.tsx:176-186`

**Issue:** The name label uses `fontSize: 20` with `fontWeight: "bold"`. A name like
"Isabella" at 20px bold will render approximately 90-100px wide — more than double the 36px
circle diameter. With 5-25 agents in close proximity on a 32px-tile grid, labels will
visually overlap neighbors. This is not a crash, but it directly affects the project's
"browser rendering smoothness with 5-25 agents" goal.

**Fix:** Reduce to `fontSize: 10` or `fontSize: 11` to keep the label compact relative to
agent density. This matches the activity text size and is consistent with standard NPC
label conventions in tile-map games.

---

### IN-02: Magic numbers in pill draw callback are disconnected from layout constants

**File:** `frontend/src/components/AgentSprite.tsx:83-88`

**Issue:** The pill dimensions `(-60, -30, 120, 16, 4)` have no relation to `RADIUS`,
`MAX_ACTIVITY_LEN`, or font size. If any of those values change, the pill must be
re-tuned manually with no static link to guide the update.

**Fix:** Extract layout constants above the component:

```tsx
const PILL_WIDTH = 120;          // px — should accommodate MAX_ACTIVITY_LEN chars at fontSize 11
const PILL_HEIGHT = 16;          // px — slightly taller than fontSize 11
const PILL_Y_OFFSET = -(RADIUS + PILL_HEIGHT + 4); // keeps pill above circle with gap
```

Then use them in `drawActivityPill` and the text position expression. No behavior change,
but future sizing changes become a single-source edit.
